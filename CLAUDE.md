Проект: Telegram-бот «Русско-болгарский переводчик с ударениями»
1. Цель проекта
Создать бесплатного Telegram-бота, который выполняет три основные функции:

Двусторонний перевод между русским и болгарским языками с сохранением разговорного стиля и автоопределением исходного языка.

Расстановка ударений в болгарских словах и текстах (на основе данных нормативного словаря).

Генерация живых разговорных примеров для заданного болгарского слова или фразы.

Бот должен быть легко развёртываем на бесплатных хостингах (PythonAnywhere, Oracle Cloud Free Tier, Koyeb и др.) и работать 24/7 с минимальным обслуживанием.

2. Функциональные требования
2.1. Общая логика
При запуске бота командой /start пользователю показывается инлайн-клавиатура с тремя кнопками:
Перевод | Ударения | Примеры

Выбранный режим сохраняется для пользователя (в памяти, базе данных или в context.user_data для вебхуков). При следующем текстовом сообщении бот действует согласно последнему выбранному режиму.

Если режим не выбран, по умолчанию используется Перевод.

2.2. Режим «Перевод»
Вход: любое текстовое сообщение (русское или болгарское).

Автоопределение языка:

Если текст на русском → перевести на болгарский.

Если текст на болгарском → перевести на русский.

Требования к переводу:

Естественная разговорная речь (никаких буквализмов и канцеляризмов).

Сохранение эмоционального окраса, идиом, фразеологизмов.

Выходной язык помечается маркером LANG:BG или LANG:RU, который затем удаляется перед отправкой пользователю.

Используемый API: OpenRouter (модель google/gemini-3-flash-preview или аналогичная, указанная в конфигурации).

Выход: чистый перевод без пояснений, отправленный обратно пользователю в Telegram.

2.3. Режим «Ударения»
Вход: болгарский текст (предполагается, что пользователь вводит уже болгарскую фразу).

Выход: тот же текст, но с проставленными знаками ударения (\u0300) над ударными гласными.

Алгоритм:

Текст разбивается на токены (слова, знаки препинания, пробелы).
Для каждого слова длиной более одной гласной выполняется запрос к нормативному словарю rechnik.chitanka.info.
Если слово найдено напрямую, берётся его форма с ударением.
Если не найдено, определяется базовая форма (lemma) и ударение переносится на ту же позицию гласной в исходном слове.
Если ударение не удалось найти ни через прямую, ни через базовую форму — слово возвращается без изменений.
Регистр букв (заглавная первая буква) сохраняется.
Требования к парсингу:

Устойчивость к изменениям структуры сайта (основной парсинг через поиск span[id^="name-stressed_"] и fallback – поиск любых кириллических токенов с комбинирующими символами ударения).

Обработка HTML-сущностей (&#x0430; и т.п.).

Таймауты запросов не более 10 секунд.

Кэширование: результат ударения для каждого уникального слова может быть закэширован в памяти или в Redis для ускорения повторных запросов.

2.4. Режим «Примеры»
Вход: болгарское слово или короткая фраза.

Выход: 3 нумерованных разговорных примера использования на болгарском языке (каждый пример на новой строке). Примеры должны быть реалистичными, из бытовой речи, без излишней формальности.

Используемый API: OpenRouter (та же модель, системный промпт для генерации примеров).

Постобработка: добавляется заголовок "Примеры:\n\n" перед списком.

2.5. Обработка ошибок
Если API OpenRouter недоступен или возвращает ошибку, пользователю отправляется сообщение: "Сервис временно недоступен. Попробуйте позже."

Если словарь ударений не отвечает, режим «Ударения» возвращает исходный текст без ударений.

При неизвестном режиме или отсутствии текста – игнорировать или отправлять короткую подсказку.

3. Нефункциональные требования
Бесплатность эксплуатации:
Бот должен работать на бесплатных или условно-бесплатных хостингах (PythonAnywhere, Oracle Cloud Always Free, Koyeb, Railway с осторожностью и т.п.).
Расходы на OpenRouter API: используется бесплатный тариф (если есть) или ключ с ограничением. В спецификации указывается, что API-ключ предоставляется администратором бота.

Асинхронность:
Все сетевые вызовы (Telegram API, OpenRouter, словарь ударений) должны быть асинхронными (aiohttp, asyncio).

Масштабируемость:
Бот рассчитан на одного-нескольких пользователей одновременно. Для хранения режимов достаточно context.user_data (при вебхуках) или словаря в памяти. При необходимости масштабирования (более 1000 пользователей) следует добавить Redis или SQLite.

Простота развёртывания:
Проект должен содержать requirements.txt, пример файла .env и подробное README с инструкцией по установке на PythonAnywhere, Oracle Cloud и локально.

5. Взаимодействие с внешними API
5.1. Telegram Bot API
Используется через python-telegram-bot.

Методы: sendMessage, sendChatAction, setWebhook, answerCallbackQuery.

Формат вебхука: POST https://<your-domain>/webhook.

5.2. OpenRouter API
Эндпоинт: https://openrouter.ai/api/v1/chat/completions

Заголовки: Authorization: Bearer <ключ>, Content-Type: application/json

Модель: по умолчанию google/gemini-3-flash-preview (может быть изменена через переменную окружения).

Системные промпты:

Для перевода: (описан выше).

Для примеров: краткий запрос на генерацию 3 болгарских разговорных примеров.

Температура: 0.5 (перевод), 0.6 (примеры).

5.3. Словарь ударений (rechnik.chitanka.info)
Формат: https://rechnik.chitanka.info/w/<слово_без_ударения>

Метод: GET, возвращает HTML-страницу.

Парсинг:

Поиск span с атрибутом id="name-stressed_*".
Если не найдено – поиск всех кириллических слов с комбинирующими символами (U+0300/U+0301) и выбор наиболее частотного варианта, совпадающего по написанию без ударения.
Для производных слов – извлечение ссылки на базовую форму (производна форма ... <a href="/w/...">).
Повторный запрос для базовой формы и перенос ударения на исходное слово по индексу ударной гласной.

7. Варианты развёртывания (спецификация для opencode)
В README должны быть представлены как минимум три способа запуска:

7.1. Локальный запуск (тестирование)
Клонировать репозиторий.

Установить зависимости: pip install -r requirements.txt

Создать файл .env с TELEGRAM_BOT_TOKEN и OPENROUTER_API_KEY.

Запустить: python bot.py (long polling).

7.2. Развёртывание на PythonAnywhere (бесплатно, вебхук)
Зарегистрироваться, создать веб-приложение Flask.

Загрузить код в домашнюю директорию.

Установить зависимости через консоль: pip3.10 install --user ...

Настроить вебхук: https://<username>.pythonanywhere.com/set_webhook

Не забывать продлевать работу приложения раз в 3 месяца.

7.3. Развёртывание на Oracle Cloud Free Tier (VM, long polling)
Создать VM (Ubuntu 22.04).

Установить Python, git, зависимости.

Настроить systemd-сервис для автоматического перезапуска бота.

Использовать nohup или tmux для демонизации.

7.4. Развёртывание на Koyeb / Render (вебхук, с учётом «засыпания»)
Использовать Docker-образ или прямое подключение репозитория.

Установить переменные окружения.

Активировать вебхук (аналогично PythonAnywhere).

11. Критерии приёмки (для проверки реализации)
Бот отвечает на /start клавиатурой с тремя кнопками.

При выборе режима «Перевод» любой текст на русском переводится на болгарский и наоборот; перевод звучит естественно.

В режиме «Ударения» болгарский текст возвращается с правильно расставленными ударениями (хотя бы для 90% слов из частотного списка).

В режиме «Примеры» возвращается 3 различных болгарских предложения, содержащих запрошенное слово.

Бот успешно развёрнут на одном из бесплатных хостингов и работает без вмешательства в течение нескольких дней.

Код опубликован в открытом репозитории с лицензией, позволяющей свободное использование и модификацию.
